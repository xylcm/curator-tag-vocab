# 导出按钮位置调整

> GitHub Issue: #97
> 创建日期: 2026-03-05
> 状态: Draft

## 1. 概述

将导出功能按钮（Export Protobuf 和 Export CSV）从当前控制栏区域移至页面右上角，使页面布局更加清晰合理，将主要操作（筛选、搜索）与次要操作（导出）进行视觉分离。

## 2. 用户故事

- 作为标签管理员，我希望导出按钮位于页面右上角，以便快速找到数据导出入口
- 作为标签管理员，我希望导出按钮与筛选操作区域分离，以便更清晰地识别页面主要功能区域

## 3. 功能描述

### 3.1 入口与触发方式

导出按钮始终可见，位于页面右上角区域，与页面标题和统计信息同处一个视觉层级。

### 3.2 核心流程

**当前布局：**
1. 页面顶部：标题 + 统计信息
2. 控制栏：搜索框 + 筛选条件下拉框 + Load Tags 按钮 + Add Tag 按钮 + Export Protobuf 按钮 + Export CSV 按钮
3. 标签列表区域

**调整后布局：**
1. 页面顶部：标题 + 统计信息 + **导出按钮组（右上角）**
2. 控制栏：搜索框 + 筛选条件下拉框 + Load Tags 按钮 + Add Tag 按钮
3. 标签列表区域

### 3.3 界面与交互

**导出按钮组位置：**
- 位于页面标题栏（header）的右侧
- 与标题 "Curator Tag Vocabulary" 处于同一水平线
- 两个导出按钮并排显示，保持原有样式（btn-info）

**按钮顺序：**
- 从左到右：Export Protobuf | Export CSV

**响应式适配：**
- 桌面端：按钮水平排列在标题栏右侧
- 移动端（可选）：按钮可折叠或保持紧凑排列

**交互行为：**
- 点击 Export Protobuf：下载 protobuf 格式的标签数据文件
- 点击 Export CSV：下载 CSV 格式的标签数据文件
- 按钮点击后的加载状态和成功/失败提示保持不变

## 4. 验收标准

- [ ] Export Protobuf 和 Export CSV 按钮显示在页面标题栏的右上角区域
- [ ] 按钮从控制栏（controls）中移除
- [ ] 按钮保持原有的视觉样式（btn-info）和交互行为
- [ ] 按钮点击后能够正常触发导出功能
- [ ] 页面其他功能（搜索、筛选、添加标签等）不受影响
- [ ] 在常见屏幕分辨率下布局显示正常
