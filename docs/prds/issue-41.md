# 导出按钮位置调整

> GitHub Issue: #41
> 创建日期: 2025-02-24
> 状态: Draft

## 概述

将导出按钮（Export Protobuf 和 Export CSV）从当前的筛选操作区域移动到页面右上角，与标题水平对齐，以优化页面布局并提升用户体验。

## 当前行为

导出按钮（"Export Protobuf" 和 "Export CSV"）目前位于页面中部的筛选控制区域（`.controls-wrapper`），与其他筛选条件（搜索框、分类筛选、排序选项等）以及操作按钮（"Load Tags"、"Add Tag"）混在一起。

这种布局的问题：
- 导出功能属于全局操作，与筛选条件混在一起，视觉层级不清晰
- 用户需要滚动或浏览整个筛选区域才能找到导出功能
- 页面顶部标题区域右侧空间闲置，未被有效利用

## 期望行为

将两个导出按钮移动到页面右上角，与标题 "Curator Tag Vocabulary" 水平对齐：
- 按钮位于标题右侧，靠右展示
- 按钮保持原有样式和功能（蓝色 info 样式）
- 按钮顺序：Export Protobuf 在左，Export CSV 在右
- 与标题保持垂直居中对齐
- 保持响应式布局，在窄屏幕下合理适配

目标效果示意：
```
[Curator Tag Vocabulary]                                    [Export Protobuf] [Export CSV]
```

## 验收标准

- [ ] 导出按钮从 `.controls` 区域移除
- [ ] 导出按钮显示在页面右上角，与标题水平对齐
- [ ] 按钮保持原有样式（蓝色 info 样式）
- [ ] 按钮点击功能正常工作（导出 Protobuf 和 CSV 文件）
- [ ] 页面在桌面端和移动端均正常显示，无布局错乱
