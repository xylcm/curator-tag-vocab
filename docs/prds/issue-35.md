# 导出按钮布局调整

> GitHub Issue: #35
> 创建日期: 2025-01-21
> 状态: Draft

## 概述
将导出按钮（Export Protobuf 和 Export CSV）从当前的控制栏区域移动到页面右上角，与页面标题水平对齐并靠右展示，使界面布局更加清晰合理。

## 当前行为
导出按钮（"Export Protobuf" 和 "Export CSV"）目前位于页面中部的控制栏（controls）区域内，与其他筛选条件、操作按钮（如 Load Tags、Add Tag）混在一起。这导致：
- 控制栏区域按钮过多，视觉拥挤
- 导出功能作为全局操作，与筛选/搜索等局部操作混在一起，层级不清晰

## 期望行为
导出按钮应位于页面右上角，与页面标题 "Curator Tag Vocabulary" 水平对齐并靠右展示：
- 导出按钮与标题在同一水平线上
- 按钮靠右对齐，与页面右侧保持适当间距
- 按钮样式保持不变（保持原有的 `.btn-info` 样式）
- 两个导出按钮保持原有的顺序和间距

## 验收标准
- [ ] 导出按钮（Export Protobuf、Export CSV）显示在页面右上角
- [ ] 导出按钮与标题 "Curator Tag Vocabulary" 水平对齐
- [ ] 导出按钮靠右展示
- [ ] 按钮原有的样式和交互行为保持不变
- [ ] 原有控制栏区域不再显示导出按钮
- [ ] 页面在不同屏幕宽度下布局正常（响应式适配）
