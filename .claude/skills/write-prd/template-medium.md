# 中等需求 PRD 模板

适用于：对现有功能的增强，涉及部分 UI 或交互变更（如新增筛选项、新增导出格式）

```markdown
# {功能名称}

> GitHub Issue: #{ISSUE_NUMBER}
> 创建日期: {YYYY-MM-DD}
> 状态: Draft

## 1. 概述
{这个功能解决什么问题，给用户带来什么价值}

## 2. 用户故事
- 作为 [角色]，我希望 [做什么]，以便 [达到什么目的]

## 3. 功能描述

### 3.1 入口与触发方式
{用户如何发现和进入该功能}

### 3.2 核心流程
{编号步骤描述主流程，每步包含用户操作和系统响应}

### 3.3 界面与交互
{新增或修改的 UI 元素、视觉状态、交互细节}

## 4. 验收标准
- [ ] 用户应该能够 ...
- [ ] 当 ... 时，应该 ...
```
