# 复杂需求 PRD 模板

适用于：全新功能或根本性变更，涉及新用户流程、多个 UI 组件或跨功能影响

```markdown
# {功能名称}

> GitHub Issue: #{ISSUE_NUMBER}
> 创建日期: {YYYY-MM-DD}
> 状态: Draft

## 1. 概述
{这个功能解决什么问题，给用户带来什么价值}

## 2. 用户故事
- 作为 [角色]，我希望 [做什么]，以便 [达到什么目的]

## 3. 功能描述

### 3.1 入口与触发方式
{用户如何发现和进入该功能}

### 3.2 核心流程
{编号步骤描述主流程，每步包含用户操作和系统响应}

### 3.3 界面与交互
{新增或修改的 UI 元素、视觉状态、交互细节}

### 3.4 数据展示
{展示哪些信息、排序分组规则、空状态展示}

## 4. 边界情况
{数据为空、操作失败、极端数据量、并发冲突等异常场景}

## 5. 验收标准
- [ ] 用户应该能够 ...
- [ ] 当 ... 时，应该 ...
- [ ] ... 应该在 ... 内完成
```
