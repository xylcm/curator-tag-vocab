# 重构技术方案模板

适用于：代码重构或技术优化，无 PRD，直接从 Issue 描述出发

```markdown
# {重构标题} — 技术方案

> GitHub Issue: #{ISSUE_NUMBER}
> 创建日期: {YYYY-MM-DD}
> 状态: Draft

## 1. 概述

{1-2 句话说明重构的目标和动机}

## 2. 现状分析

{描述当前代码的问题：架构缺陷、技术债、性能瓶颈等，引用具体文件和代码}

## 3. 目标状态

{描述重构后的代码结构和预期改善}

## 4. 重构策略

{描述渐进式的重构路径，确保每一步系统都保持可工作状态}

1. {步骤描述 — 每步可独立提交和验证}
2. {步骤描述}

## 5. 影响面分析

| 文件/模块 | 变更类型 | 说明 |
|-----------|---------|------|
| {文件路径} | 新增/修改/删除 | {变更说明} |

## 6. 测试策略

- **现有测试**: {确认哪些现有测试可覆盖重构变更}
- **新增测试**: {需要补充的测试}
- **验证方法**: {如何确认重构前后行为一致}

## 7. 风险与待决事项

- {已知风险或需要进一步讨论的决策}
```
