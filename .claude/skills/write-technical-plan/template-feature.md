# Feature 技术方案模板

适用于：新功能开发或现有功能增强，PRD 已通过评审

```markdown
# {功能名称} — 技术方案

> GitHub Issue: #{ISSUE_NUMBER}
> PRD: docs/prds/issue-{ISSUE_NUMBER}.md
> 创建日期: {YYYY-MM-DD}
> 状态: Draft

## 1. 概述

{1-2 句话说明实现目标，引用 PRD 中的核心需求}

## 2. 方案设计

### 2.1 整体思路

{简要描述技术方案的核心思路和架构选择}

### 2.2 详细设计

{按模块/层级展开具体的设计细节：接口定义、数据模型变更、组件结构、关键算法等。使用代码片段辅助说明}

## 3. 影响面分析

| 文件/模块 | 变更类型 | 说明 |
|-----------|---------|------|
| {文件路径} | 新增/修改/删除 | {变更说明} |

## 4. 实现步骤

{按依赖顺序排列，每步可独立提交和验证}

1. {步骤描述}
2. {步骤描述}

## 5. 测试策略

- {测试类型（单元/集成/E2E）}: {测试内容和验证点}

## 6. 风险与待决事项

- {已知风险或需要进一步讨论的技术决策}
```
