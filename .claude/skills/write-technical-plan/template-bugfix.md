# Bug 修复技术方案模板

适用于：缺陷修复，无 PRD，直接从 Issue 描述出发

```markdown
# {Bug 标题} — 修复方案

> GitHub Issue: #{ISSUE_NUMBER}
> 创建日期: {YYYY-MM-DD}
> 状态: Draft

## 1. 问题描述

{简要描述 Bug 的表现、影响范围和复现条件}

## 2. 根因分析

{定位到具体的代码位置和逻辑错误，说明为什么会产生这个 Bug}

**关键代码位置**:
- `{文件路径}:{行号}` — {问题说明}

## 3. 修复方案

{描述修复思路和具体的代码变更}

## 4. 影响面分析

| 文件/模块 | 变更类型 | 说明 |
|-----------|---------|------|
| {文件路径} | 修改 | {变更说明} |

## 5. 测试策略

- **回归测试**: {确保修复不引入新问题}
- **Bug 验证**: {确认原始 Bug 已修复的测试用例}

## 6. 风险与待决事项

- {已知风险或需要进一步讨论的决策}
```
